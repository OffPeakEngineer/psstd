## Problem

Theme selection is nice, but the dashboard currently carries many palette options. That can be fun, but it may be more surface than the tool needs, and every option becomes something to keep legible across states.

## Done when

- Palette list is reviewed against readability and project tone
- Low-value or hard-to-read palettes are removed or moved behind a deliberate choice
- High contrast and light/dark remain supported
- README or UI copy does not over-explain the palette system
